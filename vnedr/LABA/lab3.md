# 🔧 Лабораторная работа №3  
## Диагностика работоспособности персональных компьютеров

---

## 📋 Цель работы
Провести диагностику компьютера с помощью программ-диагностиков для проверки конфигурации ПК, работоспособности устройств и выявления возможных дефектов.

---

## 🎯 Задачи
- Проверить конфигурацию компьютера (память, диски, устройства)
- Протестировать работоспособность компонентов ПК
- Выявить «намевающиеся» дефекты оборудования
- Предотвратить возможную потерю данных

---

## 🛠 Используемое программное обеспечение

### PassMark BurnInTest
Программа для испытания компьютера «на прочность».

Возможности программы:
- ✅ Тестирование стабильности и надёжности ПК
- ✅ Синхронное распределение нагрузки на все подсистемы
- ✅ Проверка скорости процессора
- ✅ Тестирование оперативной памяти
- ✅ Диагностика жестких дисков и CD/DVD приводов
- ✅ Тестирование звуковых и видеокарт
- ✅ Проверка принтеров и сетевых устройств

Диагностируемые компоненты:
- Системная плата
- Память (стандартная, дополнительная, расширенная)
- Видеоподсистема
- Жесткие диски
- Приводы флоппи-дисков
- Клавиатура
- Порты (последовательные и параллельные)
- Координатные устройства
- CD-ROM и SCSI-устройства

---

## 📝 Порядок выполнения работы

### 1. Установка программы
1. В сетевой папке выбрать файл bitstd.exe2. Установить программу BurnInTEST
3. Запустить программу
### 2. Запуск тестов
Через меню Quick Tests доступны различные виды тестов:

- 🔥 Maximum CPU temp - максимальная температура процессора
- 💻 CPU coverage - загрузка процессора
- 🧠 RAM - тестирование оперативной памяти
- 🎨 2D graphics - memory - тестирование 2D-графики
- 🎮 3D graphics - тестирование 3D-графики
- 💾 Local disk scan - сканирование локального диска
- 💿 Removable disk scan - сканирование съёмного диска

## 📊 Параметры для записи

Для каждого теста фиксируются:
- Cycle - номер цикла тестирования
- Operations - количество выполненных операций
- Errors - количество ошибок
- Last error Description - описание последней ошибки (если есть)
