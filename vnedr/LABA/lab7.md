# Лабораторная работа №7. Диагностика и нагрузочное тестирование ПК с использованием AIDA64

## 📋 Сведения о программном обеспечении
**AIDA64 Extreme Edition** — это профессиональный программный комплекс, предназначенный для глубокого анализа, диагностики и проверки стабильности компьютеров под управлением Windows.

**Ключевые функции:**
*   Подробный аудит оборудования и выявление системных ошибок.
*   Инструменты для нагрузочного тестирования (стресс-тесты) и отслеживания показателей датчиков.
*   Бенчмарки для оценки быстродействия системы.
*   Полная поддержка актуальных и предыдущих версий ОС Windows (включая серверные редакции).

## 🔧 Структура программы
**1. Проверка производительности компонентов (CPU, RAM, Disk)**
*   **Суть:** Измерение скорости вычислений и обмена данными.
*   **Особенности:** 64-битные тесты, проверка пропускной способности ОЗУ и таймингов, поддержка современных инструкций (AVX, SSE и др.), многопоточность (до 32 потоков).
*   **Накопители:** Тесты скорости для HDD, SSD, USB-флеш и оптических приводов.

**2. Нагрузочное тестирование (Stress Test)**
*   **Цель:** Создание экстремальной нагрузки для проверки надежности.
*   **Задачи:** Поиск нестабильности при разгоне, проверка системы охлаждения и питания.
*   **Виды тестов:** Нагрузка на диски (HDD/SSD), вычисления на видеокарте (OpenCL GPGPU).

**3. Мониторинг системных датчиков**
*   **Возможности:** Считывание данных с более чем 250 сенсоров.
*   **Отслеживаемые параметры:** Температурный режим, вольтаж, обороты кулеров, энергопотребление.
*   **Вывод данных:** Трей, OSD-меню, гаджеты, экраны клавиатур Logitech, лог-файлы (совместимость с RivaTuner).

**4. Аппаратная диагностика**
*   **Плюсы:** Получение точных сведений о «железе» без физического доступа внутрь корпуса.
*   **База знаний:** Идентификация более 150 тысяч устройств.
*   **Детали:** Текущие частоты CPU, состояние мониторов, диагностика через нагрузку.

**5. Анализ программной среды**
*   **Отчетность:** Подробные данные об установленном софте (лицензии, защиты, настройки системы).
*   **Процессы:** Список активных задач, служб, драйверов, автозагрузки и истории браузера.

## 📝 Порядок выполнения работы

**1. Инсталляция и запуск**
*   Выполните установку **AIDA64 Extreme Edition** на учебный ПК.
*   Запустите приложение через ярлык на рабочем столе.

**2. Активация лицензии**
*   Перейдите в меню: `Справка` → `Ввести ключ продукта`.
*   Используйте лицензионный ключ из файла «Ключи».

**3. Изучение интерфейса**
*   Ознакомьтесь со всеми разделами утилиты.
*   Проведите сравнительный анализ возможностей AIDA64 с аналогами (PC Wizard, Furmark, Passmark, Burning Test).
*   **Результат:** Запишите выводы в рабочую тетрадь.

**4. Фиксация температурных показателей**
*   Откройте раздел: `Компьютер` → `Датчики`.
*   Запишите начальные температуры процессора и накопителей.
*   После проведения теста GPGPU зафиксируйте изменения.

| Компонент | Температура до теста (°C) | Температура после GPGPU (°C) |
| :--- | :--- | :--- |
| Процессор (CPU) | | |
| Жесткий диск (HDD) | | |
| Видеокарта (GPU) | | |

> 💡 **Важно:** Если данные недоступны в AIDA64, воспользуйтесь утилитами CoreTemp или Gputemp.

**5. Оценка производительности (Benchmark)**
*   Перейдите во вкладку `Тест`.
*   Запустите все доступные виды тестов.
*   Сравните полученные баллы с глобальной базой данных (мин/макс значения).

| Название теста | Ваш результат | Макс. результат в базе | Мин. результат в базе |
| :--- | :--- | :--- | :--- |
| | | | |

**6. Сведения о системной логике**
*   Раздел: `Системная плата` → `Чипсет`.
*   Зафиксируйте данные о микросхемах Северного и Южного мостов.

**7. Нагрузочные тесты через меню Сервис**
*   Откройте вкладку `Сервис`.
*   Выберите инструменты: `Тест GPGPU` и `Тест стабильности системы`.

**A. Бенчмарк видеокарты (GPGPU)**
*   Нажмите `Start Benchmark`.
*   Внесите данные о температуре и напряжении в таблицу:

| Параметр | GPU | CPU |
| :--- | :--- | :--- |
| | | |

**B. Проверка стабильности (Stability Test)**
*   Установите длительность теста: **10 минут**.
*   Нажмите `Start`.
*   После завершения перейдите во вкладку `Statistics`.
*   Запишите минимальное и максимальное значение температуры процессора за период теста.

## ❓ Вопросы для закрепления материала

1.  **Какова главная цель проведения стресс-тестирования?**
    *   Оценка устойчивости системы при пиковых нагрузках.
    *   Диагностика проблем с охлаждением, питанием или разгоном.
    *   Проверка надежности компонентов перед активной эксплуатацией.

2.  **Зачем применяются диагностические утилиты?**
    *   Для получения сведений о конфигурации ПК без разборки корпуса.
    *   Для поиска программных конфликтов и аппаратных неисправностей.
    *   Для (мониторинга) температур и напряжений в реальном времени.

3.  **Какой функционал охватывает программа AIDA64?**
    *   Полную диагностику оборудования.
    *   Проведение бенчмарков и стресс-тестов.
    *   Мониторинг датчиков и анализ установленного ПО.
