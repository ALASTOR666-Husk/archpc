# Практическая работа №7
## Тестирование компьютерной системы в программе AIDA64

---

## 📋 Общая информация о AIDA64

**AIDA64 Extreme Edition** — мощный комплекс диагностики и тестирования Windows-систем для домашних пользователей.

### Основные возможности:
- Широкий спектр инструментов для обзора и помощи в диагностике ошибок
- Стресс-тесты и мониторинг датчиков системы
- Уникальные технологии оценки производительности
- Совместимость со всеми версиями Windows (включая Windows 8 и Windows Server 2012)

---

## 🔧 Основные разделы программы

### 1. Тестирование процессора, памяти и дисков

**Характеристики:**
- Набор **64-битных тестов** для измерения производительности
- Тестирование пропускной способности оперативной памяти и задержек
- Использование инструкций: **MMX, SSE, XOP, FMA, AVX, AVX2**
- Поддержка до **32 потоков**
- Тестирование: HDD, SSD, оптических дисков и флеш-накопителей

### 2. Стресс-тесты

**Назначение:**
- Создание максимальной нагрузки на систему
- Тестирование стабильности и надежности
- Выявление проблем стабильности

**Доступные тесты:**
- Жесткого диска
- SSD
- OpenCL GPGPU

### 3. Мониторинг температуры, напряжения и вентиляторов

**Возможности:**
- Поддержка **250 различных датчиков**
- Отображение:
  - Температуры
  - Напряжения
  - Скорости вращения вентиляторов
  - Потребления электроэнергии

**Отображение данных:**
- В системном трее
- OSD-панели
- Гаджете рабочего стола
- Дисплее клавиатуры Logitech G15/G19
- Журнал или экспорт (RivaTuner, Samurai)

### 4. Диагностика

**Преимущества:**
- Точные данные о компонентах системы **без вскрытия компьютера**
- База данных: **более 150,000 устройств**
- Определение текущей частоты процессора
- Статус CRT/LCD монитора
- Выявление проблем через стресс-тесты

### 5. Анализ ОС и прикладных программ

**Информация:**
- 50 страниц данных об установленных программах
- Лицензии, антивирусные программы
- Настройки ОС
- Запущенные процессы, сервисы, DLL-файлы
- Программы в автозапуске
- Посещенные веб-страницы

## 📝 Задание

### Этапы выполнения:

1. **Установка программы**
   - Установить AIDA64 Extreme Edition на рабочий компьютер

2. **Запуск программы**
   - Использовать значок на рабочем столе

3. **Активация**
   - Ввести ключ продукта (из файла "Ключи")
   - Путь: Справка → Ввести ключ продукта

4. **Анализ функционала**
   - Изучить весь функционал программы
   - Сравнить с: **PC Wizard, Furmark, Passmark, Burning Test**
   - Записать вывод в тетрадь

5. **Сбор информации о температуре**
   - Вкладка: **Компьютер → Датчики**
   - Зафиксировать температуру:
     - Центральный процессор
     - Жесткие диски
   - Оформить в виде таблицы

   | Устройство | t до теста | t после теста GPGPU |
   |------------|------------|---------------------|
   | CPU        |            |                     |
   | HardDisk   |            |                     |
   | GPU        |            |                     |

   > 💡 **Примечание:** Если информация не отображается, использовать программы **CoreTemp** и **Gputemp**

6. **Бенчмарк тест**
   - Перейти на вкладку тест

7. **Сравнение результатов**
   - Провести каждый вид теста
   - Занести результаты в таблицу и сравнить с базой данных

   | Название теста | Ваш результат | Максимальный результат | Минимальный результат |
   |----------------|---------------|------------------------|----------------------|
   |                |               |                        |                      |

8. **Информация о чипсете**
   - Вкладка: **Системная плата → Чипсет**
   - Записать информацию о микросхемах Северного и Южного мостов

9. **Перейти на вкладку Сервис**

10. **Тест GPGPU**
    - Вкладка: **Сервис**
    - Выбрать: **тест GPGPU и тест стабильности системы**

11. **Benchmark GPGPU**
    - Нажать **Start Benchmark**
    - Записать таблицу по температуре и напряжению

    | Тест | GPU | CPU |
    |------|-----|-----|
    |      |     |     |

12. **Тест стабильности**
    - Длительность: **10 минут**
    - Нажать клавишу **Start**
    - Перейти на вкладку **Statistics**
    - Записать минимальную и максимальную температуру процессора

## ❓ Контрольные вопросы

1. Для чего необходимо выполнять **стресс тесты** компьютерной системы?

 - Проверки стабильности системы под максимальной нагрузкой
 - Выявления перегрева, нестабильного питания или разгона
 - Тестирования надёжности компонентов перед эксплуатацией

2. Для чего используются **программы диагностики** компьютера?

 - Сбора информации о железе и ПО без вскрытия ПК
 - Выявления ошибок, конфликтов и неисправностей
 - Мониторинга температур, напряжений и производительности в реальном времени

3. Какая из известных программ обладает **наибольшим функционалом** для тестирования ПК?

 - Диагностику оборудования
 - Бенчмарки и стресс-тесты
 - Мониторинг датчиков
 - Анализ ОС и установленного ПО
